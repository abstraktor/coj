Background
A virtual machine provides hardware abstraction. Although pretty transparent, it leaks when looking at performance.

Building a performance-optimized requires knowledge about the „outer“ environment of the language system and it’s own performance characteristics. After doing application-specific performance-optimizations such as removing unnecessary event bindings, developers start optimizing with respect to outer-language performance characteristics, such as Google Chrome JIT-ing only methods smaller than a maximum block size.

Those environment-specific optimizations may be useless in different environments or even conflict between various products leading to tradeoffs between environment systems.

Solution
I use COP to enable programmers speeding up their applications with the awareness of how the code gets translated into processor instructions. The language runtime will enable or disable layers depending on the hardware and other virtualization layers it is running on.

It needs to be determined if the layers should be characterized by the name and version of the products involved or by the optimization features that are implemented in the environment. Furthermore, it is open which VM-layers those optimizations could depend on.

In this seminar, I will (a) investigate into what outer characteristics are used for such optimizations, (b) suggest useful layers based that and (c ) implement a performance optimization that illustrates the use of such layers.

Impact
With COP-leak, developers can write environment-optimized programs in a maintainable way without needing to decide between platforms.
