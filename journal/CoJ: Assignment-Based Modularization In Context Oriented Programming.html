I present a framework for changing the behavior of a J program without loosing the overridden behavior.

Users are able to describe layers of functionality by refining some assignments and then running the original algorithm with the modified code.

Object-oriented programs as well as few-architectured scribbles can facilitate that in a similar way.


Motivation
We found authors of object-oriented J programs using comments, booleans or immediate functions to enable or disable particular features of the program. With CoJ, we suggest a unified, semantic, well-readable way of formulating feature switches. Furthermore changing a feature switch at runtime becomes feasible and visible.

Small scribbles can take advantage of layering when trying alternatives. Authors can use a new layer to start an experiment with alternating data or the algorithm the scribble runs with. When working this way, duplicating code to start a thinking branch can be avoided and a denser, more readable working style of ideation becomes possible.
